---> Descrição do Projeto
Este projeto foi desenvolvido como parte de um desafio da plataforma **DIO (Digital Innovation One)**. O objetivo é criar um pipeline de **ETL (Extract, Transform, Load)** que simula o dia a dia de um Cientista de Dados.

---> O Desafio e a Solução
Devido à indisponibilidade momentânea da API original, utilizei a **Alternativa 1** do desafio: a simulação de dados via Python Lists. Essa abordagem demonstra resiliência técnica e mantém o foco na arquitetura do fluxo de dados.


### Fluxo ETL

1. Extract (Extração)
Os dados foram simulados em uma estrutura de lista de dicionários, representando informações de clientes e seus saldos.

2. Transform (Transformação)
Foi criada uma lógica de negócio para gerar recomendações personalizadas: clientes com saldo maior recebem dicas Premium, enquanto outros recebem incentivos à poupança.

3. Load (Carregamento)
Os dados transformados são exibidos em um relatório estruturado, simulando a entrega final do processo.
